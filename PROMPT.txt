写一个番茄钟应用，技术栈 React + Vite。极简设计，主要功能就计时 + 任务管理，所有数据放 localStorage。

番茄钟基本逻辑：25 分钟工作 + 5 分钟休息为一个循环，4 个循环后长休息 15 分钟（可配置）。用户可以自定义时长（工作时长、短休息、长休息、长休息前的循环数）。

计时器状态机：空闲 → 工作中 → 短休息 → 工作中 → 短休息 → ... → 长休息 → 工作中。工作中可以暂停/继续/跳过当前阶段/重置。计时精度到秒。

后台标签页：用户切到别的标签页或者最小化窗口时，setInterval 会被浏览器节流（最低 1 秒一次），用 Web Worker 或者 Page Visibility API + 时间戳补偿来保证计时准确。系统休眠唤醒后自动同步时间。

通知：阶段切换时：
- 浏览器桌面通知（Notification API）
- 声音（可配置，关闭 / 简单 beep / 自定义音频）
- 页面标题变化（显示剩余分钟）

任务管理：
- 添加任务：标题、预计番茄数、所属项目、优先级
- 任务状态：待办、进行中（选一个任务"开始"）、已完成
- 番茄完成时（工作阶段结束）自动累加到当前任务的"已完成番茄数"
- 任务列表按状态分组（待办/进行中/已完成）
- 拖拽排序优先级
- 任务可以删除（删除前确认）、编辑、归档（不显示在主列表里）

统计：
- 今日完成番茄数、累计完成番茄数
- 每日趋势（最近 30 天折线图）
- 任务完成率（已完成 / 总番茄数）
- 各类别/项目番茄数分布（饼图）
- 最专注时段（一天中哪个小时完成最多番茄）

设置：
- 番茄时长自定义
- 主题（亮/暗/跟随系统）
- 声音开关
- 自动开始下一阶段
- 严格模式（工作阶段不响应暂停请求？或者必须点两次确认）
- 每日目标（每天完成几个番茄）

数据导入导出：所有数据（任务、设置、历史）可以导出 JSON 文件，导入恢复。多个设备用同一个浏览器配置文件可以同步（通过 Chrome 自己的同步就行，不接云端）。

键盘快捷键：空格开始/暂停、R 重置、N 跳过当前阶段、Ctrl+Enter 完成当前番茄、Esc 取消。

代码组织：src/components（Timer、TaskList、TaskItem、Stats、Settings）、src/hooks（useTimer、useNotification、useWorker）、src/stores（useTimerStore、useTaskStore、useSettingsStore）、src/utils（time、date、format）、src/workers（timerWorker.js）。

依赖：react、react-dom、zustand、date-fns、@dnd-kit/sortable、recharts。≤ 8 个。
